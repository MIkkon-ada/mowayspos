# AGENTS.md｜博维 AI 升级项目驾驶舱前端重构说明

## 项目背景

这是"博维 AI 升级项目驾驶舱"的前端相关说明文件。系统的业务口径以三份外部主文档为准：

- `博维 AI 驾驶舱项目三层结构与字段口径编写计划.docx`
- `博维 AI 驾驶舱项目生命周期与权限动作口径.docx`
- `博维AI驾驶舱的角色与权限口径.docx`

核心流程是：

```text
公司 CEO 创建项目入口
→ 配置企业教练、PM、统筹人、成员和协助人
→ 项目进入项目内管理阶段
→ PM 组织启动会并拆解重点工作
→ PM 派发关键任务
→ 责任人提交汇报，协助人补充协作
→ PM 确认入库，必要时上报企业教练
→ 项目结束、复盘、归档
```

## 当前技术状态

当前前端仍以全局脚本加载为主，历史结构中存在按页面拆分后的模块文件。

不要贸然改成完全不同的加载方式，因为部分页面或模板仍可能依赖全局函数。

## 已对齐的业务主线

- 三层主结构固定为 `项目 -> 重点工作 -> 关键任务`
- 过程记录不作为第四层主结构
- 公司 CEO 负责项目入口配置和全局查看
- 企业教练是单项目内最高把关角色
- 项目负责人 / PM 负责日常推进、确认入库和闭环
- 关键任务责任人和协助人需要分开表达

## 拆分原则

1. 不要重写业务逻辑。
2. 不要改变接口路径。
3. 不要改变现有页面结构和样式。
4. 不要改变数据库字段名。
5. 不要改变现有状态值。
6. 每次只拆一个页面或一个模块。
7. 拆完必须做回归检查。
8. 所有会被页面调用的函数必须保持可访问。
9. 如果函数被多个页面共用，不要强行移动。
10. 不确定的函数先保留在原位置，并在报告里说明。

## 权限原则

所有权限最终以后端为准，前端隐藏按钮只是辅助。

角色包括：

- 公司 CEO
- 企业教练
- 项目负责人 / PM
- 项目统筹人
- 关键任务责任人
- 关键任务协助人

关键规则：

- 公司 CEO 只做入口配置和全局查看。
- 企业教练负责把控方向、重大事项和结束复盘把关。
- 项目负责人 / PM 负责启动会、任务派发、汇报确认和入库。
- 项目统筹人只能协调和建议，不默认拥有最终确认权。
- 责任人负责执行并提交正式汇报。
- 协助人负责补充协作内容，不替代责任人。

## 中文文案与编码规则

1. 禁止把中文文案替换为 `?`、乱码、拼音或英文占位符。
2. 所有用户可见文案必须保持中文。
3. 如果终端或写入链路会污染中文，必须使用 Unicode escape 写法保存中文字符串。
4. 修改 JS 文件后必须执行 `node --check <file>`。
5. 修改用户界面文案后必须检查页面是否出现乱码。
