# PROJECT_STATE

## 项目目标

- 当前项目是博维 AI 驾驶舱的主实现仓。
- 主业务口径以三份外部文档为准：
  - `博维 AI 驾驶舱项目三层结构与字段口径编写计划.docx`
  - `博维 AI 驾驶舱项目生命周期与权限动作口径.docx`
  - `博维AI驾驶舱的角色与权限口径.docx`
- 当前仓库内的摘要文档只做落地和同步，不重新定义业务规则。

## 当前业务主线

```text
项目
  └─ 重点工作
      └─ 关键任务
```

流程主线：

- 公司 CEO 创建项目入口并配置人员
- 企业教练接手项目内把关
- 项目负责人 / PM 组织启动会、拆解重点工作、派发关键任务
- 责任人提交汇报，协助人补充协作信息
- PM 确认入库，必要时上报企业教练
- 项目结束后复盘并归档

## 技术栈

- 后端：FastAPI
- 数据库：SQLite + SQLAlchemy
- 前端：独立 `frontend/` 项目
- 业务数据来源：SQLite 种子数据 + 导入脚本

## 关键目录

- `app/main.py`：FastAPI 入口
- `app/models.py`：项目、重点工作、关键任务以及相关业务模型
- `app/routers/`：各模块 API
- `app/services/extractor.py`：AI 提取模拟逻辑
- `frontend/src/App.tsx` / `frontend/src/pages/` / `frontend/src/components/`：核心前端页面渲染与交互
- `frontend/src/styles.css`：全局样式
- `frontend/index.html`：前端入口
- `app/excel_importer.py`：Excel 导入逻辑
- `bowei_ai_dashboard.db`：当前本地数据库

## 重要决策

- 三层主结构固定为 `项目 -> 重点工作 -> 关键任务`。
- 过程记录不作为第四层主结构。
- 公司 CEO 只做全局管理和项目入口配置，不做项目日常推进。
- 企业教练是单项目内最高把关角色。
- 项目负责人 / PM 负责日常推进、确认入库和闭环。
- 关键任务责任人和协助人要分开表达，不能只用“成员”一个入口。
- 旧字段名可以保留兼容映射，但新页面应优先使用新口径。
